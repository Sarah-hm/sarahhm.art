Principles

Determined Chaos

The deterministic nature of chaos produces seemingly random patterns. An environment and components that allow some level of entropy inform a serendipitous space.

  • Is this chaos useful? (Being as useful as your favorite song can suffice)
  • What does it subvert? What are its affordances?

Cybernetic

The interaction between the human and the machine (and human-human, machine-machine) must be clear. Feedback loops happen in playful and non-invasive ways.

  • Is this interaction playful and/or meaningful to the experience?

Accessible

Accessible design, both in the form and in the content, must coexist with the user’s experience. A responsive interface, a semantically legible DOM, and optional complex interactions will allow greater experience customization according to users’ desires.

  • Can the user experience be improved and/or simplified?
  • Is the chaos (or maximalist nature) taking over?
  • Is the designed hierarchy improving accessibility? If not, could it be improved?

    Sizing

    /* sizing */
    font-size: 10px;
    /* p-size unless otherwise specified */
    --p-size: clamp(1rem, 0.956vw + 0.694rem, 2rem);
    --h1-size: clamp(5rem, 12.428vw + 1.023rem, 18rem);
    --h2-size: clamp(3rem, 2.868vw + 2.082rem, 6rem);
    --h3-size: clamp(2.5rem, 0.478vw + 2.347rem, 3rem);
    --h4-size: clamp(1rem, 0.956vw + 0.694rem, 2rem);
    --menu-font-size: clamp(1.5rem, 0.956vw + 1.194rem, 2.5rem);
    --button-font-size: clamp(1.5rem, 0.956vw + 1.194rem, 2.5rem);
    --figcaption-font-size: calc(var(--p-size) / 3 * 2);
    --sup-font-size: 0.5em;

    All font size are defined using clamp functions, creating an automatic responsive design wherein the minimum, maximum, and desired values are calculated using the viewport width.

    For instance, the H1 heading has a minimum value of 5rem (1rem=10px) and a maximum value of 18rem according to the viewport width.

    Spacing

    /* spacing */
    --section-gutter: clamp(2rem, 2.868vw + 1.082rem, 5rem);
    --inline-grid-gap: clamp(2.5rem, 31.071vw + -7.443rem, 3.5rem);
    --block-grid-gap: clamp(3rem, 2.008vw + 2.358rem, 5.1rem);
    --inner-padding-large: 1.5em;
    --inner-padding: 0.5em;
    --button-padding: var(--inner-padding);

    Wide negative spaces, namely side gutters and grid gaps, are also defined with clamp functions using a minimum and maximum value.

    Smaller negative spaces, namely component paddings, paragraph margins, line-hegihts, are defined using em. Ems unit measures are dependent on the parent component, making the child component responsive to whichever font-size the parent is set to.

    Grids use Media queries' breakpoints to collide into fewer columns, all reaching 1 column grid templates at the smallest breakpoint (768px).
